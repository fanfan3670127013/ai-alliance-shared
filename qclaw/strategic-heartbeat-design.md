## 🎯 设计理念

**"让系统自己说话，指挥官只需决策"**

战略心跳不是简单的定时任务，而是帝国OS的"生命体征监测仪"：
- **主动感知**：不等问题爆发，提前发现端倪
- **智能分级**：区分紧急/重要/常规，精准推送
- **行动导向**：每条心跳都带可执行建议

---

## 💓 心跳机制

### 心跳类型
| 类型 | 触发频率 | 推送目标 | 内容深度 |
|------|---------|---------|---------|
| **微心跳** | 每5分钟 | 系统日志 | 核心指标快照 |
| **标准心跳** | 每30分钟 | 指挥官 | 完整状态报告 |
| **深度心跳** | 每2小时 | 指挥官 | 含预测分析 |
| **紧急心跳** | 实时触发 | 指挥官 | 立即需处理 |

### 紧急心跳触发条件
- Agent离线 > 5分钟
- 任务阻塞 > 30分钟
- 系统可用率 < 95%
- 风险评分骤降 > 20分

### 自动触发机制
```javascript
// Cron表达式
const CRON_STANDARD = "0 */30 * * * *";    // 每30分钟
const CRON_DEEP = "0 0 */2 * * *";          // 每2小时

// QClaw心跳执行逻辑
async function executeHeartbeat(type = "standard") {
  const report = await collectSystemMetrics();  // 采集指标
  const analysis = await analyzeDeviation(report);  // 偏差分析
  const suggestions = await generateSuggestions(analysis);  // 生成建议
  
  // 推送报告
  await pushHeartbeatReport(report, type);
  
  // 执行自动任务（仅紧急项）
  if (analysis.hasCritical) {
    await executeEmergencyActions(analysis);
  }
}
```
