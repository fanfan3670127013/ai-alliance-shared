## 🎯 设计理念

**"不只看结果，要理解原因"**

因果预测引擎是帝国OS的"决策显微镜"：
- **向后看**：从历史事件中挖掘因果规律
- **向前看**：预测当前Action的未来影响
- **选优解**：基于预测推荐最优行动路径

---

## 🔬 核心能力

### 能力1：因果图谱构建
基于历史数据自动构建事件因果图谱：
- **节点**：系统事件（任务完成、模块上线、参数变更）
- **边**：因果关系（强度 = 相关性 × 时序性）
- **权重**：边权重表示因果强度（0-100%）

```
因果图谱示例：
┌──────────────┐
│ 启动监控面板  │ ──[强度: 85%]──→ ┌──────────────┐
└──────────────┘                   │ 问题发现率↑ │
                                    └──────────────┘
                   ┌───────────────────────────────[强度: 72%]──→ ┌──────────────┐
                   │                                               │ 决策效率↑    │
                   │                                               └──────────────┘
                   ▼
            ┌──────────────┐                    ┌──────────────┐
            │ 双Agent协同  │ ──[强度: 90%]──→ │ 开发效率↑    │
            └──────────────┘                   └──────────────┘
```

### 能力2：假设推演
给定期望的Action，预测系统状态变化：


找到对目标影响最大的"杠杆点"：

```

│
└─ 最优选择: 路径1 (协同频率)
    预期收益: +22% 开发效率
```

## 🔧 技术实现方案

### 因果图谱数据结构
```javascript
// 因果节点
const causalNode = {
  id: "node_001",
  type: "action",      // action | outcome | state
  label: "启动监控面板",
  category: "system",   // system | process | result
  createdAt: "2026-05-17T02:00:00",
  weight: 1.0
};

// 因果边
const causalEdge = {
  source: "node_001",
  target: "node_002",
  strength: 0.85,       // 0-1 表示因果强度
  type: "direct",      // direct | indirect | temporal
  evidenceCount: 5,     // 支持该因果关系的证据数
  lastUpdated: "2026-05-17T02:00:00"
};

// 假设推演请求
const hypothesisRequest = {
  action: "启动战略心跳",
  timeHorizon: "7d",    // 预测7天内的影响
  context: {
    currentState: { ... },
    constraints: ["计算资源<10%", "响应时间<500ms"]
  }
};

// 假设推演响应
const hypothesisResponse = {

};
```

### 预测算法逻辑
```javascript
async function predictCausalEffect(action) {
  // 1. 从图谱中查找直接影响的节点
  const directNodes = findDirectEffects(action, causalGraph);
  
  // 2. 计算直接影响
  const directEffects = directNodes.map(node => ({
    metric: node.label,
    change: node.weight * confidenceFactor,
    confidence: calculateConfidence(node)
  }));
  
  // 3. 递归计算间接影响（最深3层）
  const indirectEffects = [];
  for (const node of directNodes) {
    const secondOrder = findDirectEffects(node, causalGraph, { maxDepth: 3 });
    indirectEffects.push(...secondOrder);
  }
  
  // 4. 计算成本
  const costs = estimateCosts(action);
  
  // 5. 评估风险
  const risks = assessRisks(action, directEffects);
  
  // 6. 综合评分
  const score = calculateOverallScore(directEffects, costs, risks);
  
  return {
    action,
    directEffects,
    indirectEffects,
    costs,
    risks,
    overallScore: score,
    recommendation: score >= 70 ? "recommended" : "neutral"
  };
}
```

## 🎯 使用场景

### 场景1：行动前预测
**场景**：准备启动战略心跳系统  
**操作**：输入Action → 查看预测结果 → 决策是否执行  
**价值**：避免盲目决策，提前了解收益与风险

### 场景2：优化方向选择
**场景**：需要提升开发效率  
**操作**：输入目标 → 查看最优干预路径 → 执行推荐Action  
**价值**：找到投入产出比最高的优化点

### 场景3：历史经验学习
**场景**：复盘预测准确率  
**操作**：查看历史预测 → 对比实际结果 → 优化图谱权重  
**价值**：持续提升预测能力
